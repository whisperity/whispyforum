Filename and directory terminology

Introduction

Filenames and directory places are used to describe the purpose of a file and/or files in a directory.

Filename specific terminology

Prefix Suffix File extension Comment
example.   .php Example scripts
tool.   .php Tool scripts
  .class .php Classes
cached.   .php Cache scripts. (e.g: svn information files) Cached content files are usally generated while executing a scripts and deleted after execution.
cached.   .ptmp Cache pictures (e.g: uploaded pictues, omitting original file extension)

Directory specific terminology

Directory Comment
/ Main files
/documentation Documentation files.
/documentation/general General documentation files
/documentation/style-data Documetation stlyesheets and images
/documentation/globals Documenation about global variables (e.g: configuration array)
Rest of the /documentation/ directories follow the structure of the general directory structures
/includes Class and function libraries
/templates Template files
/templates/admin Administrator control panel templates
/templates/framework General framework templates (header, footer, etc.)
/templates/install Templates used by the install script
/templates/user Templates used for anything related to users (login box, user control panel)
/themes Theme data
/themes/winky Stylesheet and images for the winky template
/upload Files uploaded from the browsers of users and admins
/upload/usr_avatar User avatar pictures (both thumbnails and cached picture templates)
